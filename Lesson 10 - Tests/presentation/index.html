Typy testowania

Testy jednostkowe

  • Testowanie kod - nie wychodząc poza klasę
  • Często służy to testowania warunków krańcowych
  • Powinno być równopranym zadaniem dewelopera oprócz pisania

Testy integracyjne

  • Testowanie niskopoziomowe zależności - wykorzystanie klas pomiędzy sobą
  • Sprawdzamy czy to co napisaliśmy działa poprawnie jako trybik w aplikacji
  • Te testy dotyczą jeszcze kodu

Testy interfejsu

  • Testowanie interfejsu
  • Wykorzystujemy przeglądarkę, a nie edytor kodu
  • Na tym etapie przydaje się dydykowana rola testera

Testy systemowe (e2e)

  • Testowanie całego przekroju aplikacji
  • Wymaga przygotowania zarówno klienta jak i serwera

Test Driven Development

  • Metodyka tworzenia oprogramowania
  • Pierw test, dopiero potem kod
  • Powinno gwarantować 100% pokrycia
  • Biblioteka pozwalająca pisać testy i ich warunki
  • Nie zapewnia wykonania testów - jedynie je opisuje
  • Szeroko wykorzystywana zarówno do testów jednostkowych i interfejsu

Podstawy

  • describe
  • it
  • expect
  • toBe

Użycie podstaw biblioteki


describe('Basic laws of code', () => {
    it('should be valid', () => {
        expect(true).toBe(true);
    });
});

Porównywanie

  • toBe
  • toEqual
  • not
  • toBeTruthy, toBeFalsy
  • toBeDefined, toBeNull
  • toContain
  • toBeLessThan, toBeGreaterThan
  • toBeCloseTo

Użycie porównań

  • expect('test').toBe('test')
  • expect({ test: true }).toEqual({ test:true })
  • expect({ test: true }).not.toBe({ test:true })
  • expect(0).toBeFalsy()
  • expect(null).toBeNull()
  • expect([1,2,3,4,5]).toContain(1)
  • expect(0).toBeLessThan(1)
  • expect(3.14).toBeCloseTo(3, 0)

Łapanie wyjątków

toThrow, toThrowError


expect(() => {
    throw new Error('my error message');
}).toThrowError('my error message')

Szpiedzy

  • spyOn, createSpy, createSpyObj
  • toHaveBeenCalled, toHaveBeenCalledWith
  • and.returnValue, and.callFake

Szpiedzy w akcji


spyOn(myObj, 'myMethod')
myObj.myMethod = jasmine.createSpy('Bond drinking')
myObj = jasmine.createSpyObj('Bond', ['myMethod, 'myOtherMethod'])

expect(myObj.myMethod).toHaveBeenCalled()
expect(myObj.myMethod).toHaveBeenCalledWith('arg 1', 2)


spyOn(myObj, 'myMethod').and.callThrough()

jasmine.createSpy('myMethod').and.callFake((arg) => { return !!arg;})

myObj.mytMethod.and.returnValue(44)

Testy asynchroniczne


it('should fail before jasmine timeouts', (done) => {
    setTimeout(() => {
        expect(true).toBe(false);
        done();
    }, 1000);
})
